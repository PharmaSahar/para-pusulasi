# PROJECT 001 - Slice 3 Phase 4 Human Review Queue

Label: LOCAL ADVISORY EVIDENCE - NOT PRODUCTION VALIDATION

## Objective

Create a deterministic, append-only, advisory human-review queue that converts shadow quality findings into reviewable items without changing production behavior.

Advisory constraints in this phase:
- no automatic action
- no blocking
- no regeneration
- no upload/scheduler decision changes
- no Telegram
- no UI
- no YouTube mutation

## Architecture

Source rows:
- logs/shadow_content_quality_results.jsonl rows generated by shadow checkpoints
- validated via src/shadow_content_quality.py row validator (v1/v2 compatibility)

Queue foundation:
- src/shadow_review_queue.py

Core components:
- typed queue item model (immutable dataclass)
- typed queue event model (append-only)
- deterministic queue builder (idempotent ingestion)
- replay engine to reconstruct current state from events
- read-only query and summary helpers

Fail-open behavior:
- malformed source rows are counted and reported
- builder returns diagnostics instead of raising into generation/upload paths
- no pipeline integration is required in this phase

## Eligibility Rules

Always eligible:
- HIGH or CRITICAL findings
- financial_claim_risk with MEDIUM/HIGH confidence
- ticker/company mismatch classes
- explicitly listed high-impact finding codes

Conditionally eligible:
- MEDIUM severity findings
- low/info findings with correlated multi-signal count
- selected unsupported feature follow-up when operationally useful

Normally non-reviewable:
- safe educational/negated contextual findings
- low-confidence isolated noise
- selected unsupported-not-implemented informational findings

## Deduplication and Idempotency

Stable item identity uses:
- source evaluation_id
- finding_code
- affected_artifact
- evidence_hash
- source row schema version

Behavior:
- identical source finding maps to same review_item_id
- repeated ingestion does not create duplicate open item
- material evidence/hash change creates new item id
- prior open item version can be marked SUPERSEDED
- historical events are preserved append-only
