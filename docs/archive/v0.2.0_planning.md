# v0.2.0 Planning

## Goal

Deliver a deterministic daily opportunity report from passive research events.

Definition of Done:
- Replaying the same event window twice produces the same Top 10 output.

## Scope Boundaries

- No production publishing flow changes.
- Keep collectors passive and fail-open.
- No autonomous production decisions.
- Keep changes small, reversible, and test-covered.

## Sprint Plan

### Sprint 1: Source Expansion

1. Product Hunt collector
- Branch: `feature/product-hunt-collector`
- Deliverables:
  - `src/product_hunt_collector.py`
  - scheduler registration (passive)
  - contract-conformant raw events
- Acceptance:
  - valid `schema_version=1` events
  - collector failures do not break one-shot run
- Tests:
  - collector unit tests (happy path + fail-open)
  - scheduler integration coverage

2. Hacker News collector

### Sprint 2: Shared Fetch Layer

3. Shared fetch abstraction
- Branch: `feature/shared-fetch-abstraction`
- Deliverables:
  - reusable HTTP helper for collectors

  - normalized error envelope for logging
- Acceptance:

- Tests:

- Branch: `feature/fetch-policy-controls`

- Acceptance:

- Tests:

## Integration Rules

- Base all feature branches on `v0.2.0-planning`.
- Merge only after tests pass and PR checklist is complete.
- Keep one logical feature per PR.

## Suggested v0.2.0 Exit Criteria

- At least 5 passive sources integrated.
- Deterministic Top 10 output validated by tests.
- Daily JSON + Markdown outputs generated from replay.
- Research regression suite remains green.
